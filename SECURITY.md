# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.3.x   | :white_check_mark: |
| 1.2.x   | :white_check_mark: |
| < 1.2   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within Windows System Manager, please follow these steps:

### 1. Do NOT disclose publicly

Please do not open a public GitHub issue for security vulnerabilities.

### 2. Contact privately

Send a detailed report to: **[your-email@example.com]**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### 3. Response Timeline

- **Initial Response:** Within 48 hours
- **Status Update:** Within 7 days
- **Resolution Target:** Within 30 days (depending on severity)

### 4. Recognition

We appreciate responsible disclosure and will:
- Acknowledge your contribution in the release notes (with your permission)
- Work with you to understand and resolve the issue

## Security Best Practices

When using Windows System Manager:

1. **Always run as Administrator** only when needed
2. **Download from official sources** (GitHub Releases)
3. **Verify file integrity** using provided checksums
4. **Keep the application updated** to the latest version

## Known Security Considerations

- This application requires Administrator privileges to manage services and tasks
- Service operations can affect system stability
- Process termination can cause data loss in affected applications

Thank you for helping keep Windows System Manager secure!
